Scroll-Driven Animations

Tie any animation to scroll position. No IntersectionObserver, no scroll events, no JavaScript. Watch the gradient progress bar fill as you scroll.

Scroll this container and watch the progress bar above fill with a gradient. It's driven entirely by CSS.

The animation is tied to the nearest scroll container using animation-timeline: scroll(nearest).

You can drive any animatable property — opacity, transforms, colors, clip-path — from scroll position.

Think: parallax, reveal animations, progress indicators, sticky header transitions — all pure CSS.

The Web Animations API took years to build this. Now it's two lines of CSS.
